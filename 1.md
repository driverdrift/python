The official Python coding style guide, PEP 8  
Indentation should use 4 spaces instead of tabs.  
In Python, naming conventions mainly follow two styles:
- Snake Case (snake_case): All letters are lowercase, with words separated by underscores.
- Pascal Case (PascalCase): The first letter of each word is capitalized, with no underscores.

| Category         | Naming Format              | Example                                                  | Notes                                                                                                                                             |
| ---------------- | -------------------------- | -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| Local Variables  | snake_case                 | user_name, age, height                                   | Variables used within functions                                                                                                                   |
| Global Variables | snake_case                 | current_status                                           | Variables at the module level                                                                                                                     |
| Global Constants | ALL_UPPERCASE + UNDERSCORE | MAX_SPEED, PI                                            | Python does not enforce constants, this is a convention                                                                                           |
| Function Names   | snake_case                 | calculate_total()                                        | Preferably start with a verb                                                                                                                      |
| Method Names     | snake_case                 | get_name(), set_value()                                  | Methods typically represent actions/behaviors of a class, following the same rules as function names                                              |
| Class Names      | PascalCase                 | UserProfile, MyBank, Person, UserAccount                 | Noun form                                                                                                                                         |
| Modules          | snake_case                 | math_utils.py, file_operations.py, math.py               | Module names should be lowercase and can use underscores to separate words                                                                        |
| Package Names    | all lowercase              | math, numpy, requests, utils, dataprocessing, textparser | Package names should be short and simple, avoid underscores to prevent cross-platform import issues (unless necessary), abbreviations are allowed |
